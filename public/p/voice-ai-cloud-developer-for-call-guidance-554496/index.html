Proposal walkthrough

Real-time RingCentral call guidance with sub-500ms AI

Voice AI / Cloud Developer for Call Guidance

In short

Gergely proposes tapping RingCentral EX's real-time media stream, transcribing audio via Deepgram or Whisper, running LLM-based intent and objection detection under 500ms, and pushing guidance to the agent UI over WebSocket. Phase 1 delivers a working single-call-leg proof of concept. Phase 2 adds session state, call logging, and scale. Rate is $75/hr.

Architecture I'd propose

  • RingCentral EX — WebRTC / SIP media tap
  • Transcription — Deepgram or Whisper stream
  • LLM Inference — Intent, objection, next action
  • FastAPI + WS — Session state per call leg
  • Agent UI — Guidance pushed <500ms
  • Postgres — Call event log + eval data
Assumes: Assumes RingCentral EX account with media-stream access enabled · Assumes agent UI is a web app; native mobile changes the WS delivery layer only · Assumes one call leg per session to start — multi-leg changes state model, not the pipeline — tell me where I'm wrong and the design adapts.

Plan

  • Phase 1 - PoC, single call leg (Week 1-2) — Live guidance on one call leg — runnable, demoed
  • Phase 2 - Scale & logging (Week 3-4) — Multi-session state, Postgres logging, reconnect logic
