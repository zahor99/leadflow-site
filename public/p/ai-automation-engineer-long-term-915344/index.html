Proposal walkthrough

Hub-and-spoke agent system for business ops automation

AI Automation Engineer (Long-Term)

In short

Gergely proposes a hub-and-spoke agent architecture: each automation (email, invoices, QuickBooks, browser tasks, CRM) is an isolated sub-agent with a clean typed interface, coordinated by a central LangGraph orchestrator. The system stays flat enough to understand today and composable enough to add the tenth automation without rewriting the first nine. Rate is $95/hr with availability to start this week.

Architecture I'd propose

  • Triggers — New email, CRM webhook, Scheduled job
  • Orchestrator — LangGraph coordinator, State + task logs (Postgres)
  • Sub-agents / Tools — Email triage, Invoice + QuickBooks, Browser (Playwright), Doc pipeline (Claude / pandas)
  • Integrations — QuickBooks API, Your CRM, FastAPI / n8n triggers
Assumes: Assumes triggers arrive via email, CRM webhooks, or scheduled jobs — not a custom event bus · Assumes QuickBooks and your CRM both expose REST APIs; Playwright covers anything that doesn't · Assumes documents are PDFs or spreadsheets — if formats differ, the extraction node changes, nothing else does · Assumes cloud deployment; local-machine runner is also viable with the same architecture — tell me where I'm wrong and the design adapts.

Plan

  • Phase 1 — Foundation + first agent (Week 1-2) — Live orchestrator + one complete automation you can run today
  • Phase 2 — Core integrations (Week 3-5) — Email, invoices, QuickBooks, CRM all wired in
  • Phase 3 — Full coordination (Week 6+) — Central assistant routing across all agents end-to-end
