XGBoost + CLIP lead-scoring pipeline on Supabase and CF Workers

Build & deploy image + text lead-scoring system: XGBoost + CLIP image signal, CF Worker, GH Actions

In short

A two-signal lead-scoring system combining XGBoost on your CRM fields with CLIP image embeddings stored in pgvector. Scoring and outcome-logging endpoints live on Cloudflare Workers; retraining runs on a GitHub Actions schedule — no always-on servers. Built and documented end-to-end.

Architecture I'd propose

  • CF Worker gateway — /score + /log-outcome
  • XGBoost scorer — CRM fields → numeric score
  • CLIP embedder — Replicate pay-per-call
  • pgvector (Supabase) — embeddings + outcome logs
  • GH Actions retrain — scheduled or threshold-triggered
Assumes: Assumes CLIP via Replicate API; HuggingFace inference is a drop-in swap · Assumes Python runtime (Supabase Edge Function or lightweight worker) handles XGBoost inference · Assumes reference image set is provided before Phase 1 ends · Assumes Supabase free/pro tier is acceptable for storage and vector search volume — tell me where I'm wrong and the design adapts.

Plan

  • Phase 1 - Scoring API live (Week 1-2) — CF Worker + XGBoost endpoint + CLIP similarity scoring
  • Phase 2 - Outcome logging (Week 3) — /log-outcome endpoint + Supabase schema + retrain workflow
  • Phase 3 - Docs + handover (Week 4) — Deployment docs, env vars, redeploy runbook
