Proposal walkthrough

GHL Webhook Workflows for Zoom Webinar Segmentation

GoHighLevel Workflow Builder

In short

Builds four GoHighLevel workflows triggered by inbound webhooks to handle Zoom webinar registration, attended nurture, no-show re-engagement, and weekly tag reset — across 24,000+ contacts. Every workflow opens with an exclusion gate for current clients, enforces 9 AM–8 PM send windows, and uses custom field variables for personalized URLs. Phase 1 delivers all four working workflows within two weeks.

Architecture I'd propose

  • Inbound Webhook — Zoom fires payload to GHL
  • Exclusion Gate — "6 mts current clients" branch
  • Field + Tag Write — Join URL → custom field; tag applied
  • Send Window Check — 9 AM–8 PM local, no Sundays
  • Nurture Sequence — SMS plain text + email, timed waits
  • Monday Tag Reset — Bulk remove mc-attended / mc-no-show
Assumes: Zoom webhook payload maps one contact per event with a unique join URL field · Approved copy deck is provided before build — step count affects hour estimate · Send-window enforcement uses GHL native time conditions, not external scheduling · Weekly reset webhook fires before any Tuesday registrations are processed — tell me where I'm wrong and the design adapts.

Plan

  • Phase 1 - Four Workflows (Week 1–2) — All four GHL workflows live, tested, documented
  • Phase 2 - Ongoing Support (Week 3+) — New sequences, modifications, troubleshooting
