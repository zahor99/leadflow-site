Proposal walkthrough

Pilot app extracting compliance matrices from government PDFs

AI Data Extraction from Large Documents

In short

Gergely proposes a focused pilot that ingests 100–300 page government PDFs and outputs a structured compliance matrix at 97%+ requirements completeness. The architecture uses semantic + structural chunking, Claude with JSON-schema structured output, embedding-based deduplication, and a built-in eval harness to measure recall before sign-off. Delivered in two phases over roughly four weeks at $95/hr.

Architecture I'd propose

  • PDF Ingest — FastAPI upload endpoint
  • Semantic Chunker — Section + clause boundaries
  • Claude Extractor — JSON schema, SHALL/MUST/SHOULD
  • Dedup + Filter — Embedding near-duplicate check
  • Taxonomy Grouper — LLM requirement-type labels
  • Matrix Output — Excel / CSV / Next.js view
Assumes: Assumes PDFs are text-layer PDFs or Word exports, not scanned images · Assumes requirement taxonomy groups are defined or agreed during Phase 1 · Assumes Claude API access; GPT-4o is a drop-in swap if preferred — tell me where I'm wrong and the design adapts.

Plan

  • Phase 1 — Working pilot (Week 1–2) — End-to-end pipeline + eval harness measuring recall
  • Phase 2 — Hardening (Week 3–4) — Dedup tuning, grouping, final matrix export
