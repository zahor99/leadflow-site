Proposal walkthrough

Claude + Graph API pipeline into SharePoint and Power BI

Claude/copilot to Microsoft OneNote specifically

In short

Gergely proposes connecting Claude to your M365 tenant via the Graph API — pulling content from OneNote, Outlook, and Teams, processing it with Claude for formatting and insight extraction, then writing structured results back to SharePoint/Excel and triggering Power BI dataset refreshes. No native Claude-M365 connector exists; Graph API is the bridge, orchestrated in n8n or Make.com. Phase 1 delivers a working OneNote-to-SharePoint pipeline before any analytics layer is built.

Architecture I'd propose

  • M365 Graph API — OneNote, Outlook, Teams
  • n8n / Make.com — trigger, retry, error alert
  • Claude API — format, summarize, extract
  • SharePoint / Excel — structured JSON → lists
  • Power BI — REST API dataset refresh
Assumes: Assumes an Azure app registration with Graph API delegated or app permissions · Assumes Claude API access; M365 Copilot licensing audit recommended before build · Assumes Power BI workspace with a push dataset or dataflow endpoint available — tell me where I'm wrong and the design adapts.

Plan

  • Phase 1 - OneNote → SharePoint (Week 1-2) — Live pipeline: notes formatted by Claude, written to SharePoint
  • Phase 2 - Analytics layer (Week 3-4) — Insight extraction → Excel / Power BI dataset refresh
