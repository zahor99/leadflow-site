Multimodal RAG + agent platform for PI litigation

Building an AI Platform for Accident Reconstruction & PI Litigation (MVP + Long-Term Partnership)

In short

A full-stack AI platform that ingests police reports, medical records, and crash-scene photos, then runs LangGraph agents to assemble timelines, flag liability, and estimate settlement ranges — all with auditable source attribution. Built in phases: a working document pipeline first, agent reasoning layer second, polished attorney-facing UI third. Anchored by a production multimodal RAG system built for a $30B/yr multinational.

Architecture I'd propose

  • Document Ingestion — OCR + structured extraction
  • pgvector Store — Semantic retrieval + rerank
  • Vision Layer — Photo analysis + mapping
  • LangGraph Agents — Timeline, liability, settlement
  • FastAPI Backend — Traced, attributed outputs
  • Next.js Frontend — Attorney-facing UI
Assumes: Assumes documents arrive as PDFs and images — Confluence or other sources change the ingestion node only · Assumes Claude or OpenAI as the LLM backbone — swappable at the agent layer · Assumes Postgres/Supabase for the vector store; Pinecone is a drop-in alternative — tell me where I'm wrong and the design adapts.

Plan

  • Phase 1 - Document pipeline (Week 1-3) — Working RAG over your real case docs, runnable today
  • Phase 2 - Agent reasoning (Week 4-6) — Timeline + liability + settlement agents live
  • Phase 3 - UI + auth + billing (Week 7-9) — Shippable attorney-facing product
