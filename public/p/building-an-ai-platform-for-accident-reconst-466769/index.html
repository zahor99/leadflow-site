Accident Reconstruction AI Platform: RAG, Vision, Agents

Building an AI Platform for Accident Reconstruction & PI Litigation (MVP + Long-Term Partnership)

In short

This is a venture-scale legal tech MVP covering three layers: document ingestion with OCR and semantic retrieval, a computer-vision layer for crash-scene photos and mapping, and LangGraph agents that produce auditable reasoning traces attorneys can cite. Built on FastAPI, Postgres/pgvector, and Next.js — the same stack already in daily production use at a $30B/yr multinational.

Architecture I'd propose

  • Ingestion — OCR / Textract + chunking
  • Vector Store — Postgres + pgvector
  • Vision Layer — Scene photos + Mapbox
  • Agent Layer — LangGraph orchestration
  • FastAPI Backend — Auth, audit log, APIs
  • Next.js Frontend — Case dashboard + chat
Assumes: Supabase or self-hosted Postgres for vector store · Claude or OpenAI as the LLM backbone · Google Maps or Mapbox for crash-site reconstruction — tell me where I'm wrong and the design adapts.

Plan

  • Phase 1 - Document Pipeline (Week 1-3) — OCR ingestion, pgvector retrieval, cited Q&A on docs
  • Phase 2 - Vision + Agents (Week 4-6) — Scene analysis, mapping layer, LangGraph agent traces
  • Phase 3 - Frontend + Auth (Week 7-9) — Case dashboard, auth, eval harness, billing hooks
