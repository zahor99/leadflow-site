Intelligence System

In short

Gergely proposes a layered AI platform sitting on top of existing HMIS — pulling data via REST/webhook connectors, running a LangGraph intelligence engine for anomaly detection and root cause analysis, and surfacing everything through a Next.js executive dashboard, AI chat, and a WhatsApp/email assistant. The MVP delivers working code, architecture docs, DB schema, and full handover. Rate is $95/hr.

Architecture I'd propose

  • HMIS Connectors — REST/webhook → Postgres/pgvector
  • Intelligence Engine — LangGraph: anomaly → RCA → recs
  • Executive Dashboard — Next.js — KPIs, flags, task status
  • AI Chat Interface — LLM with deterministic state control
  • Messaging + Email — Twilio/Meta API via n8n routing
  • Workflow Automation — n8n — assign, notify, track, retry
Assumes: Assumes HMIS exposes REST or webhook endpoints — no direct DB access needed · Assumes Azure OpenAI is the target LLM; Claude can substitute with no structural change · Assumes multi-tenant from day one; single-hospital scope simplifies the schema — tell me where I'm wrong and the design adapts.
